# 食品科普网站项目设计文档

## 1. 项目定位

本项目定位为一个面向普通消费者的食品科普网站，核心任务是围绕食品安全、食品营养、食品热点、食品标签和食品谣言进行科学解释。

网站不以行业资讯、法规数据库、企业服务或检测技术资料为核心，而是强调将专业食品科学知识转化为普通人能够理解、判断和应用的内容。

### 一句话定位

> 一个帮普通人看懂食品安全、食品营养、食品标签与食品热点的科学科普平台，并借助AI赋能。

### 核心目标

- 帮助用户理解食品安全风险，而不是制造焦虑。
- 帮助用户读懂食品标签，而不是只看营销词。
- 帮助用户看懂食品热点，而不是被标题带偏。
- 帮助用户识别食品谣言，而不是简单给出“能吃/不能吃”。
- 建立一个结构化、可持续更新的食品科普内容平台。

## 2. 网站五大核心模块

最终保留五大模块：

```text
食品安全 / 食品营养 / 热点解读 / 标签识读 / 谣言澄清
```

## 3. 模块一：食品安全

### 模块定位

解释食品安全风险从哪里来、风险有多大、普通人应该如何降低风险。

### 主要内容方向

- 微生物污染
- 食源性疾病
- 农药残留
- 兽药残留
- 重金属污染
- 真菌毒素
- 食品添加剂
- 食品包装安全
- 食品储存与冷藏
- 家庭厨房食品安全
- 外卖与预制菜安全

### 推荐文章结构

```text
1. 问题是什么？
2. 风险来源是什么？
3. 是否常见？
4. 是否可控？
5. 普通人应该怎么做？
6. 常见误区是什么？
7. 参考来源
```

### 示例选题

- 农残检出不等于超标
- 剩菜风险真正取决于什么
- 食品添加剂为什么不是洪水猛兽
- 生食水产品有哪些风险
- 黄曲霉毒素为什么值得重视
- 外卖食品安全怎么看
- 保质期到了还能不能吃
- 冷冻食品是否更不健康
- 食品抽检不合格项目怎么看
- 预制菜到底安全吗

## 4. 模块二：食品营养

### 模块定位

用食品科学和膳食指南解释日常饮食选择，避免变成“养生号”或“减肥号”。

### 主要内容方向

- 膳食结构
- 营养素基础
- 食物多样
- 控盐、控油、控糖
- 蛋白质、脂肪、碳水化合物
- 维生素和矿物质
- 食品选择
- 外卖、食堂、便利店场景
- 营养营销词拆解
- 不同人群的一般性营养提示

### 内容边界

不建议提供：

- 疾病治疗建议
- 个性化医学营养方案
- 减肥处方
- 保健食品功效承诺
- “某食物防癌/排毒/治病”类表达

### 示例选题

- 食物多样到底怎么做到
- 一天需要多少蛋白质
- 0糖饮料是不是更健康
- 低脂食品一定好吗
- 高蛋白零食适合所有人吗
- 外卖如何吃得更均衡
- 早餐不吃会怎样
- 钠含量怎么看
- 酸奶、乳饮料、发酵乳有什么区别
- 轻食为什么不一定健康

## 5. 模块三：热点解读

### 模块定位

把食品热点事件拆解为事实、标准、风险、误区和建议。

这是网站最容易做出差异化的模块之一。

### 主要内容方向

- 食品安全事件
- 食品抽检新闻
- 标准法规更新
- 预包装食品标签新规
- 食品添加剂争议
- 预制菜争议
- 校园食品安全事件
- 短视频平台食品谣言
- 网红食品争议
- 食品企业舆情事件

### 热点解读固定结构

```text
1. 一句话结论
2. 发生了什么？
3. 争议点在哪里？
4. 涉及哪些食品科学概念？
5. 相关标准、指南或监管背景是什么？
6. 普通消费者需要担心到什么程度？
7. 哪些说法是夸大或误导？
8. 普通人可以怎么做？
9. 信息来源
```

### 与普通新闻网站的区别

普通食品网站可能只转载：

```text
某地抽检发现某食品不合格
```

本网站应进一步解释：

```text
这次不合格项目是什么？
它为什么有风险？
标准限量是什么意思？
普通消费者需要担心吗？
类似食品如何选择？
```

## 6. 模块四：标签识读

### 模块定位

帮助用户读懂食品包装上的配料表、营养成分表、营养宣称和营销词。

这是本网站最适合做成交互式特色功能的模块。

### 主要内容方向

- 配料表怎么看
- 营养成分表怎么看
- NRV% 是什么
- 0糖、无糖、低糖有什么区别
- 高蛋白食品真的高蛋白吗
- 低脂食品一定更健康吗
- 钠含量怎么看
- 保质期和贮存条件怎么看
- 数字标签怎么看
- 营销宣称如何识别

### 标签识读页面建议

不要只做成文章列表，而应做成“交互式标签解释器”。

页面形式：

```text
左侧：食品包装图片
中间：可点击高亮区域
右侧：解释卡片
```

用户点击食品包装上的区域，例如：

- 配料表
- 营养成分表
- 0糖宣称
- 高蛋白宣称
- 保质期
- 贮存条件

右侧显示对应解释。

### 标签识读交互示例

```text
用户点击：0蔗糖宣称

解释：
0蔗糖不等于0糖，也不一定代表低能量。
仍应查看营养成分表中的碳水化合物和能量。

建议：
不要只看正面宣传，应结合配料表和营养成分表判断。
```

## 7. 模块五：谣言澄清

### 模块定位

识别、拆解和解释食品谣言，帮助用户形成科学判断。

不建议只做“辟谣文章列表”，而应做成“谣言卡片库 + AI 谣言拆解器”。

### 谣言卡片结构

```text
流行说法：
“隔夜菜都会致癌”

结论：
不准确 / 需要具体情况判断

为什么：
风险主要与保存时间、温度、食物类型和卫生条件有关，
不是“隔夜”本身直接决定风险。

普通人怎么做：
及时冷藏，充分加热，高风险人群更保守。
```

### 谣言分类

- 食品添加剂恐惧
- 农残恐惧
- 转基因争议
- 隔夜菜焦虑
- 0糖饮料误解
- 蛋白粉误解
- 保健食品夸大
- “排毒”“抗癌”等伪科学表述

## 8. 与传统食品网站的关键区别

传统食品网站，例如食品伙伴网，往往更偏向食品行业门户，涵盖行业资讯、法规标准、检测技术、生产管理、企业服务、数据库等内容。

本项目不应与这类网站正面竞争“资料全”和“数据库全”，而应突出以下差异：

| 维度      | 传统食品行业网站                    | 本食品科普网站                         |
| --------- | ----------------------------------- | -------------------------------------- |
| 主要用户  | 食品企业、行业从业者、检测/法规人员 | 普通消费者、学生、关注健康饮食的人     |
| 核心内容  | 行业资讯、标准法规、数据库、培训    | 科普解释、热点拆解、标签识读、谣言澄清 |
| 语言风格  | 行业化、信息密集                    | 大众化、结构化、可视化                 |
| 核心价值  | 查资料、查法规、查数据库            | 看懂问题、判断谣言、形成食品科学素养   |
| AI 创新点 | 不一定突出                          | 个性化解释、摘要、标签识读、谣言拆解   |

### 核心差异化

```text
不是更全，而是更好懂。
不是查资料，而是解释问题。
不是堆新闻，而是拆热点。
不是制造焦虑，而是给出有依据的判断。
```

## 10. 内容组织方式

### 每篇文章建议统一包含

```text
标题
一句话结论
问题背景
科学解释
标准/指南依据
普通人建议
常见误区
参考来源
更新时间
适用范围
```

### 建议增加的内容标签

```json
{
  "title": "农残检出不等于超标",
  "category": "食品安全",
  "tags": ["农残", "标准解读", "风险评估"],
  "audience": ["普通消费者"],
  "difficulty": "入门",
  "evidence_level": "国家标准/权威机构",
  "updated_at": "2026-04-25"
}
```

### 证据等级

建议给文章加“证据等级”：

```text
A：国家标准 / 官方指南
B：WHO / FAO / Codex / 权威机构
C：系统综述 / 高质量研究
D：单篇研究
E：新闻事件 / 专家观点
```

这样用户可以知道某个结论的可靠程度。

## 11. 风险表达方式

虽然本项目不做复杂食品安全决策系统，但可以在文章层面加入轻量风险提示。

### 推荐风险标签

```text
低关注：一般消费者无需过度担心
中关注：注意购买、储存和食用方式
高关注：特殊人群应谨慎
持续关注：信息仍在更新
```

注意：这里是文章级风险提示，不是用户个性化风险判断。

```text

卡片式
轻插画
信息图表
```

```text
谣言卡片
```

## 16. 内容表达原则

### 要强调

```text

适用条件
证据来源

不确定性

```

### 要避免

```text

制造焦虑
标题党
无来源结论
```

## 18. 最终建议

本食品科普网站不应做成“大而全的食品门户”

最合适的发展方向是：

```text
五大内容模块
+
三个 AI 特色功能
+
一个交互式标签识读页面
```

### 五大内容模块

```text
食品安全
食品营养
热点解读
标签识读
谣言澄清
```

### 三个优先 AI 特色功能

```text
AI 标签识读助手
AI 热点解读助手
AI 谣言澄清助手
```

### 核心价值

```text
把专业食品信息转译成普通人能理解、能判断、能行动的内容。
```

### 最终定位

> 一个以食品科学证据为基础，结合 AI 辅助解释能力，面向普通消费者的食品安全、营养标签与食品热点科普平台。
